Authentication Implementation Review

This PR introduces a complete authentication flow with JWT-based session management. The implementation follows good patterns overall, but I've identified a few areas that need attention before merging.

Security Considerations

The token validation in src/login.tsx:19 uses jwt.verify() without error handling. This method throws JwtError when the token is malformed, expired, or has an invalid signature. Wrap this in a try/catch block to prevent unhandled exceptions from crashing the request handler.

The login endpoint at .../api/routes.go:89 has no rate limiting. Without this, the endpoint is vulnerable to brute force attacks. Consider adding middleware that limits attempts per IP.

UX Issues

The LoginForm component in src/session.ts:45 doesn't handle the initial loading state. On page refresh, users briefly see the unauthenticated UI before the session check completes. Add a loading skeleton or spinner.

What Looks Good

  • Clean separation between auth logic and UI components
  • Proper TypeScript typing with AuthError union type
  • Session refresh logic handles edge cases well
  • Good use of useAuth hook for state management

Overall: Solid foundation. Address the security items before merging.
